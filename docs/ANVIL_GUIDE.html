Anvil Framework

From equations to engineering tools — a complete guide

Anvil is a Python framework for engineering computation. Write physics as plain Python functions, wire them together, and Anvil handles units, dimensional analysis, iterative solving, parametric sweeps, and a searchable library of reusable relations.

Core Concepts

Quantity Q

A number with a physical unit. Stores SI value internally. Carries dimension for unit propagation.

T = Q(300, "K")          # explicit
T = 300 * K              # stub syntax
T.value   # → 300 (in K)
T.si      # → 300 (SI float)
T.unit    # → "K"
T.to("R") # → 540 R

Relation R

A physics equation — any Python function that accepts keyword arguments and returns a dict.

def ideal_gas(T, P, R_gas):
    return {"rho": Q(P/(R_gas*T), "kg/m^3")}

# Register as RSQ:
anvil.push(ideal_gas, domain="thermo")

System

A solvable problem — a set of Quantities (inputs) and Relations (computations) wired together.

sys = anvil.system("nozzle")
sys.add("T0", 3500, "K")
sys.use(ideal_gas)
result = sys.solve_forward()

Registry (RSQ)

SQLite-backed library of Relations (R), Systems (S), and Quantities (Q). Auto-seeded on first import.

anvil.registry.list()
anvil.registry.search("mach")
anvil.R.isentropic_ratios(M=2, gamma=1.4)
nozzle = anvil.S.rocket_nozzle.copy()

Unit System

Stub syntax (recommended)

from anvil import K, Pa, m, s, kg, mol, N, J, W, rad, deg
from anvil import km, cm, mm, g, kPa, MPa, GPa, bar, atm, psi
from anvil import kN, kJ, MJ, kW, BTU, ft, inch, in_, lb, lbf
from anvil import hr, ms, kmol, g_mol, kg_mol

T  = 300 * K           # Q(300, "K")
P  = 101325 * Pa       # Q(101325, "Pa")
v  = 340 * (m/s)       # Q(340, "m/s")
g  = 9.81 * m/s**2     # Q(9.81, "m/s^2")
rho = 1.225 * kg/m**3  # Q(1.225, "kg/m^3")

String-based (also valid)

from anvil import Q
T   = Q(300, "K")
P   = Q(10, "MPa")
mdot = Q(1.5, "g/s")       # compound units parsed automatically
vol  = Q(5, "cm**3")        # Python ** notation OK
d    = Q(1, "[LMT-2]")     # raw dimension string

Conversion

T.to("R")                 # K → Rankine
v.to("km/s")              # m/s → km/s
mdot.to("kg/s")           # g/s → kg/s
Q(1, "cm^3").to("m**3")  # volume conversion

Arithmetic propagates dimensions

F = Q(100, "N")
A = Q(0.01, "m^2")
sigma = F / A         # → Quantity(10000, "Pa")  [L-1 M T-2]
E_kin = 0.5 * Q(10, "kg") * Q(30, "m/s")**2   # → J
Note: All computations internally use SI. Display unit follows what you originally typed. q.si always gives SI float. q.value gives display-unit value.

Defining Relations

Plain function (simplest)

from anvil import Q

def isentropic_exit(T0, P0, gamma, T0_T, P0_P):
    T_exit = T0 / T0_T
    P_exit = P0 / P0_P
    a_exit = (gamma * 287 * T_exit) ** 0.5
    return {"T_exit": Q(T_exit, "K"),
            "P_exit": Q(P_exit, "Pa"),
            "a_exit": Q(a_exit, "m/s")}

sys.use(isentropic_exit)    # System.use() wraps it automatically

With @anvil.relation decorator (registers automatically)

@anvil.relation(domain="thermo", tags=["gas"])
def ideal_gas_density(T, P, R_gas):
    """rho = P / (R * T)"""
    return {"rho": Q(P / (R_gas * T), "kg/m^3")}

Block relation (multiple steps)

from anvil.relation import Relation

nozzle_block = Relation.block("nozzle_physics",
    steps=[area_fn, mach_fn, isentropic_fn],
    tags=["compressible", "nozzle"]
)

Rules for relation functions

  • Accept inputs as keyword arguments — no positional args
  • Return a dict mapping output names to values or Q() objects
  • Returning Q() objects propagates units to the result automatically
  • Inputs receive SI floats unless all inputs have known dimensions (then receive Quantity)
  • Default parameter values work normally: def fn(M, gamma=1.4):

Building & Solving Systems

Build

sys = anvil.system("heat_exchanger")

# Add inputs
sys.add("T_hot_in",  600,  "K")       # string style
sys.add("T_cold_in", 300,  "K")
sys.add("mdot_hot",  2.0,  "kg/s")
sys.add(UA=5000*W/K, cp_hot=1005*J/kg/K)   # keyword style

# Add computations
sys.use(effectiveness_ntu)
sys.use(energy_balance)
sys.use("reynolds_number")     # load from registry by name
sys.use(some_rel, map={"func_T": "T_hot_in"})  # rename input

# Override inputs later
sys.set(UA=3000)              # keeps existing unit (W/K)
sys.set(UA=Q(3000, "W/K"))    # explicit Q overrides

Inspect before solving

sys.validate()           # checks for missing inputs, duplicate outputs
print(sys.info())        # lists all quantities and relations
anvil.check(sys)         # deep inspection with dependency tree

Solve — choose your solver explicitly

# Acyclic (one-pass forward propagation)
result = sys.solve_forward()

# Coupled / iterative (Gauss-Seidel fixed-point)
result = sys.solve_gauss_seidel(relaxation=0.8, max_iter=200, rtol=1e-8)
result = sys.solve_gauss_seidel(monitor=True)   # live residuals printed

# Strongly coupled nonlinear (Newton-Raphson)
result = sys.solve_newton(max_iter=50, rtol=1e-10, verbose=True)

# Auto-selects forward vs gauss_seidel based on cycle detection:
result = sys.solve()

Use results

result.summary()                    # print table
rho = result["rho"]                 # Quantity
rho.value                           # float in display unit
rho.to("g/cm^3").value             # convert
result.to_csv("out.csv")          # export
result.to_json("out.json")

Algebraic & Iterative Solvers

All in anvil.solvers. Method names are explicit — never abstracted.

find_root — scalar f(x) = 0

from anvil import solvers

x = solvers.find_root(func, bracket=(1.0, 10.0), method="brent", tol=1e-12)
x = solvers.find_root(func, x0=5.0,              method="newton", tol=1e-12)
x = solvers.find_root(func, x0=5.0,              method="secant")
# method="auto": uses brent if bracket provided, newton if x0 provided

solve_nonlinear — system F(x) = 0

x = solvers.solve_nonlinear(func, x0,
    method="hybr",    # Powell hybrid (default) — robust
    # method="lm"   — Levenberg-Marquardt
    # method="broyden1" — Broyden update
    tol=1e-10, maxiter=200, jac=None)

minimize — scalar objective f(x)

r = solvers.minimize(func, x0,
    method="L-BFGS-B",   # gradient-based with box bounds (default)
    # method="SLSQP"    — constrained (equality + inequality)
    # method="Nelder-Mead" — gradient-free
    bounds=[(0, None), (0, 10)],
    tol=1e-8, maxiter=500, jac=None)
r["x"], r["fun"], r["success"], r["nit"]

System-level iterative solvers

MethodCallBest for
"forward"sys.solve_forward()Acyclic DAG, pure feed-forward systems
"gauss_seidel"sys.solve_gauss_seidel(relaxation=0.8)Weakly coupled loops, HVAC, heat exchangers
"newton"sys.solve_newton(rtol=1e-10)Strongly nonlinear, tightly coupled systems
monitor=True on gauss_seidel: prints live residual per iteration and stores history in sys.history(). Use viz.convergence(sys) to plot.

ODE / BVP / PDE Solvers

All in anvil.solvers. Thin wrappers around SciPy with clear method names and progress output.

solve_ode — dy/dt = f(t, y) [explicit methods]

r = solvers.solve_ode(func, t_span=(0, 10), y0=[1.0, 0.0],
    method="RK45",    # Runge-Kutta 4/5 (default) — general non-stiff
    # method="RK23"   — Runge-Kutta 2/3 — lower accuracy, fewer evals
    # method="DOP853" — Dormand-Prince 8/5/3 — high accuracy
    t_eval=np.linspace(0, 10, 500), rtol=1e-8, atol=1e-10,
    events=None, verbose=True)
r["t"], r["y"], r["sol"]  # sol is callable: sol(t) → y(t)

solve_ode_stiff — stiff dy/dt = f(t, y) [implicit methods]

r = solvers.solve_ode_stiff(func, t_span, y0,
    method="BDF",     # Backward Differentiation Formula (default) — large stiff systems
    # method="Radau"  — implicit Runge-Kutta — small stiff, higher accuracy
    jac=None,         # optional Jacobian df/dy; estimated if None
    rtol=1e-6, atol=1e-10, verbose=True)
# Use for: combustion kinetics, stiff ODEs, diffusion-reaction

solve_bvp — dy/dx = f(x, y), bc(y(a), y(b)) = 0

r = solvers.solve_bvp(func, bc, x=np.linspace(0, np.pi, 5), y_init=y0,
    tol=1e-3, max_nodes=1000, verbose=False)
r["x"], r["y"], r["sol"]   # sol(x) → y(x)  [dense output]

solve_pde_heat_1d — ∂u/∂t = α ∂²u/∂x² [Crank-Nicolson]

r = solvers.solve_pde_heat_1d(
    alpha=1e-5,                         # thermal diffusivity [m²/s]
    x_span=(0, 1), t_span=(0, 10),
    u_init=lambda x: np.exp(-100*(x-0.5)**2),
    bc_left=0.0, bc_right=0.0,          # Dirichlet; None → Neumann zero-flux
    source=lambda x, t, u: np.zeros_like(x),
    nx=100, verbose=True)
r["x"], r["t"], r["u"]   # u shape: (nt+1, nx)

RSQ Registry

SQLite database at ~/.anvil/registry.db. Seeded automatically on first import.

Explore

anvil.registry.list()                    # all RSQs
anvil.registry.list(domain="aero")      # filter by domain
anvil.registry.list(type="R")           # only Relations
anvil.registry.search("mach")           # fuzzy search across name/desc/tags
anvil.registry.info("isentropic_ratios") # detailed info with inputs/outputs
anvil.registry.export("ideal_gas")      # print source code

Use RSQs

# Direct call (no System needed)
r = anvil.R.isentropic_ratios(M=2.0, gamma=1.4)
r["P0_P"]   # → 7.824

# Load a pre-built System
nozzle = anvil.S.rocket_nozzle.copy()
nozzle.set(P0=15e6, T0=3200)
result = nozzle.solve_forward()

# Inside a System
sys.use("isentropic_ratios")
sys.use("isentropic_ratios", map={"M": "M_exit"})

Register your own RSQs

# Push a function
anvil.push(my_func, name="my_rsq", domain="thermo",
           description="My ideal gas variant", tags=["gas"])

# Update existing
anvil.update(my_improved_func, name="my_rsq")

# Remove
anvil.registry.remove("my_rsq")

One-shot solve (no System object)

result = anvil.solve(ideal_gas, T=300*K, P=101325*Pa, R_gas=287*J/kg/K)

Adapters — Wrapping External Tools

Adapters let external codes (Cantera, OpenFOAM, Fortran CFD tools, REST APIs) behave exactly like Relations inside a System.

Python library adapter (e.g., Cantera)

from anvil import Adapter, Q

def my_cantera_wrapper(fuel, oxidizer, OF, Pc):
    import cantera as ct
    # ... equilibrate, extract Tc, gamma, R_gas ...
    return {"Tc": Q(Tc, "K"), "gamma_c": gamma}

cea = Adapter("cantera_cea",
    backend="python", call=my_cantera_wrapper,
    inputs={"fuel":   {"default": "H2"},
            "oxidizer": {"default": "O2"},
            "OF":     {"default": 6.0},
            "Pc":     {"unit": "Pa", "default": 10e6}},   # SI auto-convert
    outputs={"Tc":    {"unit": "K"},
             "gamma_c": {}},
    tags=["combustion", "cantera"])

engine.use(cea)  # use exactly like any other relation

CLI adapter (e.g., SU2, OpenFOAM, Fortran codes)

cfd = Adapter("su2_aero",

Project Registry

Isolated RSQ workspace per project. Prevents cluttering the global registry while iterating.

Open a project

proj = anvil.project("heat_exchanger_study", path="./my_project")
# Creates: ./my_project/.anvil/project_heat_exchanger_study.db

Register, access, search

proj.push(my_func, domain="hx", description="Draft NTU correlation")
proj.list()
proj.search("NTU")
proj.remove("old_draft")

# Access project RSQs via proj.R namespace
result = proj.R.my_func(T_in=300, mdot=2.0)
sys.use(proj.R.my_func)

Context manager — route anvil.push() to project

with anvil.project("my_study", path="./work") as proj:
    @anvil.relation(domain="test", register=False)
    def draft_ntu(UA, C_min):
        return {"NTU": UA / C_min}
    proj.push(draft_ntu)               # goes to project, not global
    anvil.R.isentropic_ratios(M=2)     # global RSQs still accessible

# Outside with block — promote when ready
proj.promote("draft_ntu")             # moves to global registry
proj.promote_all(overwrite=True)       # promote everything
Project stores are independent SQLite databases. You can have multiple projects open simultaneously; only the context-manager-active project receives anvil.push() calls.

Full Worked Example — Rocket Nozzle

import anvil
import numpy as np
from anvil import Q, K, Pa, MPa, m, kg, J

# ── 1. Load a pre-built System ─────────────────────────────────────
nozzle = anvil.S.rocket_nozzle.copy()

# ── 2. Override inputs (LOX/LH2) ───────────────────────────────────
nozzle.set(
    P0 = 20e6,    # Pa  — chamber pressure
    T0 = 3500,    # K   — chamber temperature
    gamma = 1.20, # —
    R_gas = 520,  # J/kg/K
    A_throat = 0.005,   # m²
    A_exit   = 0.08,    # m²
    P_amb    = 0,       # Pa  — vacuum
)

# ── 3. Solve (explicit solver) ──────────────────────────────────────
result = nozzle.solve_forward()
result.summary()

# ── 4. Access results with unit conversion ──────────────────────────
print(f"Thrust  : {result['thrust'].to('kN').value:.1f} kN")
print(f"Isp     : {result['Isp'].value:.1f} s")
print(f"Exit V  : {result['V_exit'].to('km/s').value:.2f} km/s")

# ── 5. Parametric sweep — chamber pressure ──────────────────────────
sweep = nozzle.sweep("P0", np.linspace(5e6, 30e6, 8), parallel=4)
sweep.summary(outputs=["thrust", "Isp", "mdot"])

# ── 6. Sensitivity — what drives Isp? ──────────────────────────────
sens = nozzle.sensitivity(outputs=["Isp", "thrust"])
sens.summary()

# ── 7. Export ───────────────────────────────────────────────────────
result.to_csv("nozzle_result.csv")
sweep.to_csv("pressure_sweep.csv")

# ── 8. Visualize ────────────────────────────────────────────────────
from anvil import viz
viz.sweep_plot(sweep, y=["thrust", "Isp"])
viz.dependency_graph(nozzle)

Full Worked Example — Custom System with Coupling

import anvil
from anvil import Q
import numpy as np

# ── Coupled heat exchanger: T_hot_out and T_cold_out depend on each other ──

def ntu_effectiveness(NTU, C_ratio):
    eps = (1 - np.exp(-NTU*(1 - C_ratio))) / (1 - C_ratio*np.exp(-NTU*(1 - C_ratio)))
    return {"effectiveness": eps}

def heat_balance(effectiveness, C_min, T_hot_in, T_cold_in):
    Q_dot = effectiveness * C_min * (T_hot_in - T_cold_in)
    return {"Q_dot": Q(Q_dot, "W")}

def outlet_temps(Q_dot, C_hot, C_cold, T_hot_in, T_cold_in):
    T_hot_out  = T_hot_in  - Q_dot / C_hot
    T_cold_out = T_cold_in + Q_dot / C_cold
    return {"T_hot_out": Q(T_hot_out, "K"), "T_cold_out": Q(T_cold_out, "K")}

sys = anvil.system("counter_flow_hx")
sys.add("T_hot_in",  600, "K")
sys.add("T_cold_in", 300, "K")
sys.add("C_hot",  2000)   # W/K
sys.add("C_cold", 3000)
sys.add("UA",     5000)   # W/K

def compute_ntu(UA, C_hot, C_cold):
    C_min = min(C_hot, C_cold)
    C_max = max(C_hot, C_cold)
    return {"NTU": UA / C_min, "C_min": C_min, "C_ratio": C_min / C_max}

sys.use(compute_ntu)
sys.use(ntu_effectiveness)
sys.use(heat_balance)
sys.use(outlet_temps)

# Coupled loop → Gauss-Seidel with live residuals
result = sys.solve_gauss_seidel(relaxation=0.7, max_iter=200, monitor=True)
result.summary(keys=["T_hot_out", "T_cold_out", "Q_dot", "effectiveness"])

# Sweep UA
sweep = sys.sweep("UA", np.linspace(500, 5000, 10), parallel=4,
                  method="gauss_seidel", relaxation=0.7)
sweep.summary(outputs=["effectiveness", "T_hot_out", "Q_dot"])
